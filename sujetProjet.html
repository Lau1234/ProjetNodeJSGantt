Donc pour résumer:

  • Vous devez faire un mini site web qui permet d'afficher un diagramme de Gantt

  • Mais vous devez aussi voir les diagrammes de Gantt des autres groupes (vous ne pourrez pas les modifier, le serveur central ne vous y autorisera pas) qui seront mis à jour en temps réel



Je vous donne la solution pour vous connecter sur le serveur central depuis votre serveur node.js.

Il suffit d'utiliser le module "socket.io-client" qui est une dépendance de socket.io (donc déjà installé sur votre projet) :

const socket = require('socket.io-client');
let client = socket.connect('Adresse IP bientôt disponnible', {reconnect: true}); // L'adresse IP vous sera communiqué dans un mail ultérieur

client.on('connect', () => {
  console.log('connected')

  client.emit('needHelp');
});
(l'event "needHelp" permet d'avoir la liste des commandes attendu et envoyer par le serveur central. Il répondra sur l'event "info")


Pour le reste, c'est comme lorsque vous utilisez Socket.io normalement, donc vous utilisez client.emit('...', []); pour envoyer des données et client.on('...', () => {...}); pour récupérer des données.


Voici un exemple d'un objet a envoyer au serveur central :

 {
 nameService : "Demo", 
 projects : [
{

  } 
 ] 
};

Petite précision, pour le serveur central un "Service" représenté ici par la propriété "nameService" est le nom de votre service / groupe. Il doit donc être unique par groupe et ne pas changer.


Quelques infos sur les évents du central :
  • needHelp (écouté par le central): Un évent supplémentaire prévu pour vous aider à utiliser les évents du central, il n'attend aucun paramètres et vous renvoi un string
  • sendUpdate (écouté par le central): Permet la création de votre "service" (c'est votre groupe, votre application, votre site quoi) ainsi que la MAJ des données lié à votre service (donc les projets et les éléments lié a chaque projet aussi). Il attend en paramètre un objet (construit comme l'exemple ci-dessus)
  • projectUpdated (emit par le central): Permet à votre serveur Node de recevoir un tableau d'objets, chaque objet est un service (comme l'exemple ci-dessus). Ce dernier est envoyé à toutes les socket connecté au central
  • errorOnProjectUpdate (emit par le central) : Est envoyé uniquement à la socket ayant tenté une modification de données, cet event ne contient qu'une seul paramètre de type string
  • deleteService (écouter par le central): Permet de retirer votre service du central.



Si vous n'arrivez pas à modifier votre objet json retourné par mongo, il faut "transformer" votre document mongo en "pure" objet json. Ex :

monSchema.find({ projet : 'test' }, (error, docs) => {
  if (error) {
    console.log('Erreur, il faut remplacer ce console.log par un traitement pour avertir le front');
  }
  // La methode .toObject() permet de transformer un document json en pure objet json
  docs = docs.toObject();
  ...
});

Autre conseil utiliser une librairie comme moment.js pour manipuler les dates. Ex : la date de début de la "tache 1" est un timestamp, je peux facilement formatter et extraire uniquement ce dont mon framework front à besoin avec moment.js.

Attention, il est possible d'avoir uniquement un chiffre et pas un nombre ou un timestamp, c'est à vous lorsque vous récupérez les données de formatter si possible correctement !


Dernier conseil, les données que vous allez recevoir du central, il faut les traiter avec la même "méfiance" que lorsqu'on récupère des données depuis une BDD NoSQL. En NoSQL on ne vérifie pas ce que l'on enregistre, c'est à la lecture qu'il faut être vigilent. Ici avec le central, faite de même.
